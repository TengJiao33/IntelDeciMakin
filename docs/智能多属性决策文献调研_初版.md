## 2. “传统多准则决策不智能”的文献化表述

目前不宜只表述为“固定数学公式”。更稳的文献化说法是：

> 传统 MCDA/MADM 方法的核心限制并不只是公式固定，而是偏好模型、权重、阈值、价值函数、损失/遗憾/前景参数等关键决策结构高度依赖决策者、分析者或预设规则；当存在历史决策、在线评论、用户行为序列等数据时，传统方法缺乏从数据中自动学习偏好结构、参数和动态变化的能力。

这个表述有三个文献支撑点：

- Hüllermeier 与 Słowiński 的 PL-MCDA 综述将 MCDA 描述为围绕决策者互动与偏好建模的构造式过程，而 PL 则用机器学习从偏好数据中学习模型；他们明确指出手工偏好建模会形成 knowledge acquisition bottleneck。  
  来源：4OR 2024 Part I, DOI [10.1007/s10288-023-00560-6](https://doi.org/10.1007/s10288-023-00560-6)，本地开放全文 `docs/external_literature_pdfs/hullermeier_slowinski_2024_part_i_pl_mcda.pdf`。
- 同一综述 Part II 系统比较 PL 与 MCDA，并讨论二者结合，说明 MCDA 与 PL 的差异不是“谁更对”，而是“构造式决策支持”与“数据驱动泛化学习”的目标、数据、评价标准不同。  
  来源：4OR 2024 Part II, DOI [10.1007/s10288-023-00561-5](https://doi.org/10.1007/s10288-023-00561-5)，本地开放全文 `docs/external_literature_pdfs/hullermeier_slowinski_2024_part_ii_pl_mcda.pdf`。
- 本地“智能 ELECTRE III”稿件把痛点具体化到 ELECTRE III 的 indifference/preference/veto thresholds：传统 ELECTRE III 需要 DM 主观给定阈值，而 NN-based threshold detection 试图从历史数据中自动学习这些阈值。  
  来源：`docs/智能多属性决策文献/利用大学排名数据集/基于神经网络的智能ELECTRE III多属性决策方法.pdf`。

所以，“最不智能”的核心可以暂时归纳为：

1. 参数依赖：权重、阈值、价值函数、偏好函数、前景/遗憾参数常由人设定或专家 elicitation 得到。
2. 公式静态：很多方法在给定矩阵后按固定聚合/距离/优序/折衷规则运行，缺少从历史反馈中更新模型的机制。
3. 泛化不足：传统 MCDA 更强调当前决策情境下的可解释支持，不天然以训练/测试泛化为目标。
4. 动态不足：在线评论、用户行为、市场评价是时间变化数据，传统静态准则矩阵很难直接表达偏好随时间变化。
5. 验证薄弱：部分在线评论排序研究用评论数、销量、平台排名等替代真实偏好或真实购买结果，基准本身可能有偏。

## 5. 现有研究的实现路径归纳

### 5.1 路径 A：Preference disaggregation / preference learning

这条线不是简单把 NN 套到 MCDM 上，而是从“给定 assignment examples / pairwise preferences / ranking examples”中学习偏好模型。典型可学习对象包括：

- additive value function 的边际价值函数；
- 类别边界或 assignment rule；
- 价值函数平滑度、复杂度与拟合度之间的正则化权衡；
- 准则非单调性；
- 准则交互；
- 时间折扣和动态偏好。

对应本地文献链条：

- EJOR 2019：regularization + non-monotonic criteria。
- EJOR 2020：diverse additive value models + valued assignment examples。
- INFORMS JOC 2020：interacting criteria。
- EJOR 2025：temporal criteria + mRNN。

这条线的“智能”不在炫技，而在把传统 MCDA 的偏好模型参数变成可从数据中学习、可泛化、可约束、可解释的对象。

### 5.2 路径 B：Outranking 方法参数学习

老师提到的 ELECTRE/PROMETHEE 类优序法，其非智能点更具体：

- ELECTRE III 需要 q/p/v thresholds；
- PROMETHEE 需要 preference functions 与阈值；
- outranking sorting 需要 class profiles、credibility、concordance/discordance 结构；
- 这些结构传统上需要 DM/专家设定。

本地“智能 ELECTRE III”稿件已经走了最直接路线：用神经网络学习 ELECTRE III 阈值，并用多进程解决 pairwise comparison 的计算负担。外部 Applied Soft Computing 2022 的 outranking-based sorting parameter learning 与这一方向高度相关，需优先下载全文核对。

### 5.3 路径 C：Deep preference learning / 可解释神经决策模型

深度学习不是必然前沿，但它在两个位置有价值：

- 作为偏好函数、价值函数、时间折扣函数的函数逼近器；
- 作为结构化模型，把 MCDA 的单调性、偏好独立、等级有序、逻辑算子等约束嵌入网络，避免纯黑箱。

本地 EJOR 2025 的 mRNN 强调保留 MCDA 的关键性质；KBS 2020 的 interpretable neural networks 则反过来用连续逻辑和 multicriteria operators 设计可解释网络。这说明“神经网络成熟”不等于没有研究空间，关键是网络是否承担了传统方法做不到的学习任务。

### 5.4 路径 D：在线评论 + MCDM 产品排序

这条线通常有固定流程：

1. 抓取在线评论；
2. 情感分析或方面级情感分析；
3. 将评论转为属性表现值、模糊数、概率语言项或其他不确定信息表达；
4. 用 TOPSIS、TODIM、regret theory、outranking、evidential reasoning 等 MCDM 方法排序；
5. 用销量、评论数、平台排名、人工偏好或案例解释验证。

目前最需要警惕的是：很多论文的“AI”只在文本处理阶段，最终排序仍由固定 MCDM 公式完成。老师录音中“在线评论排序最后一步还不智能”的判断，与这条文献链条基本一致。

### 5.5 路径 E：MORL/RL 作为相邻技术

MORL 文献的核心问题与 MADM 有相似性：多个目标冲突，若提前用单一 scalar reward 合并，会把偏好权衡隐藏在工程师预设里，并带来解释性和用户效用问题。Hayes et al. 的 MORL guide 对这个问题讲得很清楚。

但现阶段不能把 MORL 直接说成 MADM 的成熟替代方案。MORL 是 sequential decision-making，MADM/MCDA 多数是静态 alternatives ranking/sorting/choice。二者可借鉴的连接点包括：

- reward/utility scalarization 与权重预设问题；
- preference elicitation 与 preference-based RL；
- 多目标 policy set 与决策者后验选择；
- 动态偏好与时间变化。

要继续走 RL，必须检索“preference-based RL / RLHF / MORL + MCDA/MCDM ranking/sorting”的直接交叉文献，否则容易变成概念嫁接。

## 6. 验证方式与实验设置整理

| 文献类型 | 常见数据 | 常见指标 | 风险点 |
|---|---|---|---|
| Preference learning for sorting | assignment examples、合成数据、信用/用户分类/大学排名 | accuracy、precision、recall、F-score、assignment consistency、Kendall/Spearman | 训练/测试划分是否清楚；是否过拟合；类别标签是否可靠 |
| Outranking 参数学习 | 人工 assignment examples、大规模历史分配、QS 排名 | 排序/分类准确率、阈值恢复误差、稳定性、泛化 | 参数真值通常不可观测；需要强 baseline |
| 在线评论产品排序 | PConline、手机/汽车/移动支付等在线评论 | TRD、与销量/评论数/平台排序差异、case comparison | 评论数/销量是否真能代表真实偏好；平台排名机制不透明 |
| 深度偏好学习 | 结构化准则数据、时间序列行为数据 | ML 分类指标 + MCDA 可解释性检查 | 黑箱模型可能破坏 MCDA 性质；需证明单调性/偏好结构 |
| MORL/RL | sequential environment、simulator、control tasks | hypervolume、coverage set、policy utility、regret | 与静态 MADM 任务不完全同构；需要明确状态-行动-奖励定义 |

## 7. 目前能成立的低风险判断

1. “传统 MADM/MCDA 不智能”的高质量表述应聚焦“关键偏好结构依赖人工设定、缺乏从数据自动学习与动态更新”，而不是简单说“公式固定”。
2. 文献中已经存在一条成熟的 preference learning / preference disaggregation 路线，它和 MCDA 深度相连，不是完全外部的 AI 概念。
3. 在线评论产品排序文献很多，但 AI 往往集中在评论抽取与情感分析，最终 MCDM 排序仍常由预设公式完成。
4. 本地智能 ELECTRE III 稿件与外部 outranking 参数学习文献共同说明：把优序法的阈值/参数学习化，是一个比泛泛说“AI+MCDM”更具体的切口。
5. RL/MORL 可以作为“动态、多目标、偏好反馈、奖励/效用学习”的理论参照，但目前还不能说已有充分文献支持“RL 直接替代 MADM 排序”。
