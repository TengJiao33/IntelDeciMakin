# 新增公开数据集接入记录

## 1. 本次新增目的

本次新增数据集不是为了单纯堆数量，而是补足第一轮 POC 暴露出的证据缺口：

- `education_evaluation`：补教育/评价类真实有序任务，回应“大学/机构评价之外是否还有评价场景”。
- `health_risk_evaluation`：补风险分级任务，提供临床测量 + 有序风险标签的场景。
- `toc_uco_ordinal_classification`：补更广泛的 tabular ordinal classification 泛化层，检验 APSS 是否只在少数 MCDA-like 数据上有效。

## 3. 具体处理方式

### 3.1 Student Performance

新增 4 个 entry：

- `student_performance_math_no_prior_grades`
- `student_performance_math_with_prior_grades`
- `student_performance_portuguese_no_prior_grades`
- `student_performance_portuguese_with_prior_grades`

标签来自最终成绩 `G3`，转为五级有序类别：

- `G3 <= 9` -> `Cl1`
- `10 <= G3 <= 11` -> `Cl2`
- `12 <= G3 <= 13` -> `Cl3`
- `14 <= G3 <= 15` -> `Cl4`
- `G3 >= 16` -> `Cl5`

`no_prior_grades` 不使用 `G1/G2`；`with_prior_grades` 使用 `G1/G2`。二者必须分开看，因为 `G1/G2` 会显著降低任务难度。

### 3.2 Higher Education Students Performance

新增 1 个 entry：

- `higher_education_students_performance`

源数据的 `GRADE` 为 0-7，处理为 `Cl1`-`Cl8`。列 `1`-`30` 作为模型准则，`COURSE ID` 只保留为元数据。

### 3.3 Maternal Health Risk

新增 1 个 entry：

- `maternal_health_risk`

标签映射为：

- `high risk` -> `Cl1`
- `mid risk` -> `Cl2`
- `low risk` -> `Cl3`

因此更大的 `class_index` 表示更低风险/更优健康状态。临床测量值按原方向 min-max 归一化，未强行转换为 gain-type 准则。

### 3.4 TOC-UCO

新增 22 个 entry，只处理 `metadata.csv` 中 `is_oc=True` 的原始 ordinal classification 数据集，不处理 `is_oc=False` 的离散化回归任务。

处理规则：

- 所有特征 min-max 归一化为 `g1`, `g2`, ...
- 标签 `y` 按源数据的排序类别映射为 `Cl1`, `Cl2`, ...
- TOC-UCO 用于泛化鲁棒性，不作为 MCDA-specific 证据层。

## 6. 对论文设计的含义

这次新增后，数据层可以分成四层：

1. `synthetic_structure_recovery`：验证结构发现是否正确。
2. `monotone_classification` + `polish_research_units` + `qs_university_rankings`：MCDA-like 主证据层。
3. `education_evaluation` + `health_risk_evaluation` + `wine_quality`：应用扩展层。
4. `toc_uco_ordinal_classification`：泛化鲁棒层。

后续正式实验不应把所有层混在一起平均，而应分层报告。否则 TOC-UCO 的数量会压过 MCDA-specific 数据，导致研究叙事从“自适应决策”漂移成一般 ordinal classification。
