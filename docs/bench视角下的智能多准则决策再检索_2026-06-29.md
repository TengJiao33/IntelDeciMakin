# Bench 视角下的智能多准则决策再检索

日期：2026-06-29  
触发问题：online reviews MADM 的核心短板不是“方法不够复杂”，而是缺乏可信外部 benchmark；没有 ground truth 时，排序一致性和差异解释很容易变成自说自话。

## 1. 先把问题说清楚

你提出的判断是对的：online reviews 产品排序天然很难有强 bench。

- 如果用在线评论本身做排序依据，那么评论是后验、局部、带选择偏差的主观表达。
- 如果引入价格、销量、评论数、平台排名，这些变量也受渠道、营销、曝光、库存、品牌、价格策略影响，不是纯产品质量 ground truth。
- 如果只比较 TOPSIS、VIKOR、TODIM、均值法等排序一致性，只能说明“没有乱排得太离谱”，不能证明新方法更优。
- 如果用差异解释说明“我排得更合理”，审稿人仍然可以认为这是 case-based justification，不是 benchmark-based evidence。

因此，要提升说服力，最直接的不是继续在 online reviews MADM 里堆复杂模型，而是把研究任务改造成“有可评测目标”的决策问题。

## 2. 近年文献基本怎么做 bench

### 2.1 把 ranking 改成 sorting/classification

这是最常见、也最容易建立 bench 的方式。不要直接问“产品 A 应该排第几”，而是问“这个对象应分到哪个有序类别”。类别标签可以来自历史事实、专家标注、平台等级、风险等级、质量等级。

典型形式：

- 输入：多个准则/属性。
- 输出：有序类别或 assignment example。
- 指标：accuracy、AUC、0/1 loss、precision、recall、F-score、Kendall/Spearman、assignment consistency。
- baseline：logistic regression、SVM、MLP、rule/tree models、UTADIS、Choquet、TOPSIS/PROMETHEE/ELECTRE 变体。

文献例子：

- Martyn & Kadziński 的 Deep Preference Learning for MCDA 用 threshold-based sorting model，并在 benchmark problems 上用 AUC 和 0/1 loss 比较，DOI [10.1016/j.ejor.2022.06.053](https://doi.org/10.1016/j.ejor.2022.06.053)。
- Li et al. 的 temporal criteria preference learning 用 mobile game user value classification 做真实案例，并与 ML、DL、传统 MCDA baseline 比较，arXiv [2309.12620](https://arxiv.org/abs/2309.12620) / EJOR 2025。
- Li, Zhang & Pedrycz 的 Omega 2025 用大规模 assignment examples 学 non-monotonic preferences，并用 red wine quality classification 与人工/真实数据实验验证，DOI [10.1016/j.omega.2024.103219](https://doi.org/10.1016/j.omega.2024.103219)。

这条路线的优点：bench 清楚。缺点：研究对象会从“产品排序”变成“对象分级/分类”，但这反而更像可发表的实验型决策智能研究。

### 2.2 用公开机器学习数据集，把特征解释成准则

很多 preference learning / MCS 论文会用公开数据集做 benchmark。常见做法是：

- 把 features 当作 criteria；
- 把 class label 当作 assignment；
- 用 train/test 划分验证泛化；
- 对比传统 ML 与 MCDA-inspired models。

可用场景包括：

- wine quality：质量等级天然有序，适合 non-monotonic preference；
- bank marketing：是否订阅/客户响应，可转为 preference/choice；
- credit rating / default risk：风险等级或违约标签；
- medical diagnosis：甲状腺癌、乳腺癌、疾病风险等级；
- education/research units evaluation：机构等级、排名等级；
- game/user value classification：用户价值等级。

文献线索：

- Data-driven MCDM for thyroid cancer diagnosis 把诊断问题做成数据驱动 MCDM，DOI [10.1007/s10479-018-3093-7](https://doi.org/10.1007/s10479-018-3093-7)。
- Omega 2025 non-monotonic MCS 明确使用 red wine quality classification，并在 artificial and real-world datasets 上做统计检验，DOI [10.1016/j.omega.2024.103219](https://doi.org/10.1016/j.omega.2024.103219)。

这条路线数据获取最简单，但要注意：不能只说“我把 UCI 数据集套到 MADM”，必须解释为什么这个任务确实是多准则排序/分级问题。

### 2.3 用实验室人类偏好数据构造 ground truth

这是比 online reviews 更干净的“消费者偏好”路线：自己设计决策任务，让真实受试者做 pairwise comparisons 或完整排序，再用模型重建其偏好。

近年例子：

- Jiang et al. 2025 把 pairwise comparisons、response time、attention duration 结合到 Bayesian preference learning；实验是 mobile phone contract selection，30 名真实受试者，使用带时间、眼动、鼠标追踪组件的应用，验证模型重建完整偏好的能力。arXiv [2504.14938](https://arxiv.org/abs/2504.14938)。

这条路线的 bench 非常清楚：真实受试者偏好就是要重建的目标。优点是说服力强；缺点是需要实验设计、伦理/招募、系统采集。

### 2.4 用动态决策场景，验证能否追踪偏好变化

这条线比静态 online reviews 更前沿。问题不是“产品最终排序是什么”，而是“DM 的偏好随环境变化时，模型能否追踪并预测”。

近年例子：

- Zhao et al. 的 EJOR 2026/2025 accepted paper 用 linear Gaussian state space model 做 dynamic preference learning，场景包括 military context 和 computational experiments，目标是捕捉 time-varying contingent preferences，DOI [10.1016/j.ejor.2025.08.008](https://doi.org/10.1016/j.ejor.2025.08.008)。
- Grillo, Kotłowski & Kadziński 的 EJOR 2025 把 ordinal regression 与 online learning 结合，做 interactive preference learning for multiple criteria choice and ranking with provable guarantees，DOI [10.1016/j.ejor.2025.05.045](https://doi.org/10.1016/j.ejor.2025.05.045)。

这条路线的 bench 是动态任务中的预测/重建/后悔界/查询效率。优点是比在线评论新；缺点是问题抽象程度更高，应用数据需要设计。

### 2.5 学准则交互或非单调偏好

近年很多智能 MCDA 不再只学权重，而是学：

- marginal value functions；
- non-monotonic preference；
- criteria interaction；
- thresholds；
- attention-based interaction importance。

近年例子：

- Information Fusion 2026 的 attention network + threshold-based value-driven sorting procedure，用 monotonic blocks 近似边际价值函数，用 attention network 捕捉准则交互，DOI [10.1016/j.inffus.2025.103443](https://doi.org/10.1016/j.inffus.2025.103443)。
- Omega 2025 的 PLNN/PLFMNN 处理大规模 assignment examples 和 non-monotonic preferences，DOI [10.1016/j.omega.2024.103219](https://doi.org/10.1016/j.omega.2024.103219)。

这条路线和“智能 ELECTRE III 学阈值”属于同一类思想：不是换一个排序公式，而是把传统模型中最依赖人工的部分变成可学习对象。

## 3. 更适合我们找 bench 的场景候选

| 场景 | bench 强度 | 数据获取 | 决策味道 | 风险 |
|---|---:|---:|---:|---|
| wine quality / product quality classification | 高 | 很容易 | 中 | 可能太像普通 ML，需要强调有序类别与准则解释 |
| credit risk / credit rating | 高 | 中等，公开数据多 | 高 | 金融特征解释和伦理需谨慎 |
| medical diagnosis / risk stratification | 高 | 中等，公开数据多 | 高 | 医疗高风险，需避免临床结论过度 |
| university ranking / institution evaluation | 中高 | 可获得 | 高 | QS/THE 的标签本身是商业排名，机制不完全透明 |
| research unit / project evaluation | 中高 | 较难 | 很高 | 公开数据不一定充足 |
| mobile game / app user value classification | 高 | 难，除非有公开/合作数据 | 中高 | 数据不易复现 |
| 手机套餐/服务方案选择实验 | 高 | 可自己采 | 很高 | 需要受试者实验和系统设计 |
| military / emergency / supply-chain dynamic decision | 中高 | 较难 | 很高 | 真实数据难，可能只能仿真 |
| online reviews product ranking | 低到中 | 已有积累 | 中 | 最大问题是外部 ground truth 不稳 |

## 4. 对 online reviews 的重新定位

如果继续用 online reviews，更合理的定位可能不是“产品最终排序更好”，而是：

1. 把在线评论作为 criteria extraction / sentiment measurement 的输入；
2. 把最终任务改成有标签的 classification/sorting，例如产品是否高满意、是否高复购、是否高退货风险；
3. 或者做 preference reconstruction：让受试者基于评论摘要/属性信息做 pairwise choice，再看模型能否重建人类偏好；
4. 或者把评论作为一个模态，和销量、价格、售后、评分、退货等多源数据共同预测外部标签。

这样 online reviews 还可以留下，但不再承担“自己证明自己排序正确”的重担。

## 5. 暂时结论

目前近年文献给出的信号很明确：

- online reviews MADM 的最终排序验证确实是系统性软肋；
- 更强的发表路径往往把任务改成有 ground truth 的 sorting/classification/preference reconstruction；
- 2025/2026 前沿集中在 dynamic preference、interactive/online learning、non-monotonic preferences、criteria interaction、attention/NN 结构化偏好学习；
- 最容易落地的 bench 不是“找到一个客观产品排名”，而是选择一个本身有标签的数据集或自己构建人类偏好实验。

如果后续继续推进，下一步应先做“benchmark inventory”：列出 20 个候选数据集，按标签质量、准则解释性、样本量、任务新颖性、是否适合 MCDA 结构化模型打分，而不是先选模型。
