## 0. 与录音原始问题的对齐

根据 `docs/新录音-音频_原文 (6).docx`，老师最初提出的问题不是“做一个通用 AI 框架”，也不是单纯“找 benchmark 跑 accuracy”，而是：

> 传统 MADM/MCDA 方法中存在大量需要决策者人工设定的参数、偏好函数和结构假设；在智能时代，应利用 AI 技术从数据中自动学习这些人工设定，从而实现智能多属性决策。

录音中最直接的样板是智能 ELECTRE III：

- 传统 ELECTRE III 需要人工设定 q/p/v thresholds；
- 决策者往往不知道这些阈值如何确定；
- 团队已有工作尝试用神经网络学习 q/p/v；
- 用大学排名数据验证排序效果；
- 实验验证必须有明确指标，例如 accuracy、precision、F1 等。

因此，本实验设计必须始终锚定这个原始问题：

> APSS 不是凭空提出一个普通模型选择器，而是把“传统 MADM 参数智能化”推进到“传统 MADM 偏好结构智能化”。

这里的偏好结构包括：

- 数值参数，例如 q/p/v、类别阈值、风险偏好系数；
- 函数选择，例如 preference function、边际价值函数形状；
- 结构假设，例如单调性、非单调性、准则交互；
- 模型复杂度，例如是否需要 full flexible model。

所以，APSS 的实验不能只证明“分类更准”，还必须证明：

- 它减少了人工设定；
- 它能选择或拒绝复杂结构；
- 它比固定结构参数学习更稳；
- 它仍然输出可解释的 MADM 偏好结构。

## 1. 我们到底要做什么

当前研究不应表述为“提出一个更复杂的 MADM 模型”，而应表述为：

> 在有标签多准则分级/排序任务中，构建一个验证门控的自适应偏好结构选择框架，使模型能够在单调、非单调、交互和简单加性结构之间选择最小充分结构。

这里的“自适应”不是无限制涌现，而是限定在一个可解释的偏好结构空间中进行选择。

### 1.1 研究对象

输入：

- 多个备选对象 alternatives；
- 每个对象在多个准则上的表现 `g1, g2, ...`；
- 有序类别标签 `class_index`，数值越大表示越优；
- 可选外部排序标签，例如 QS rank。

输出：

- 对新对象的有序类别预测；
- 一个可解释的偏好结构：
  - 哪些准则保持单调；
  - 哪些准则允许非单调；
  - 哪些准则对存在交互；
  - 类别阈值如何划分；
  - 模型复杂度是否必要。

### 1.2 核心研究问题

RQ1：不同多准则分级任务是否真的需要不同偏好结构？

RQ2：一个验证门控的结构选择框架，能否在不显著牺牲预测性能的情况下，选择更紧凑、更稳定的偏好结构？

RQ3：在有结构真值的 synthetic benchmark 上，该框架能否恢复单调、非单调和交互结构？

RQ4：在真实 benchmark 上，该框架是否能解释“何时应该复杂，何时不该复杂”？

## 2. 我们不做什么

为了避免研究失控，第一阶段不做以下事情：

- 不做完整 self-evolving agent；
- 不做开放式结构涌现；
- 不声称发现真实客观偏好；
- 不把 QS 或其他外部排名当成绝对真值；
- 不把所有传统 MADM 方法都复现一遍；
- 不以纯 black-box neural network 作为主贡献；
- 不把 online reviews 产品排序作为主 benchmark。

## 3. 方法对象：候选结构空间

第一版框架应从一个小而硬的结构空间开始。

### 3.1 候选模型家族

| 代号 | 模型 | 结构含义 | 作用 |
|---|---|---|---|
| M0 | Majority / Prior | 只预测训练集中最常见类别 | sanity baseline |
| M1 | Linear Additive | 线性加性得分 + 类别阈值 | 最简单可解释模型 |
| M2 | Monotone Additive | 单调边际函数 + 加性聚合 + 类别阈值 | 对应传统单调 MCDA 假设 |
| M3 | Non-monotone Additive | 非单调边际函数 + 加性聚合 + 类别阈值 | 检验非单调偏好是否必要 |
| M4 | Sparse Interaction | 单调加性基础 + 稀疏 pairwise interactions | 检验准则交互是否必要 |
| M5 | Full Flexible | 非单调边际函数 + 稀疏交互 | 最复杂可解释模型 |
| M6 | APSS | Adaptive Preference-Structure Selection | 我们的结构选择框架 |

### 3.2 APSS 的最小定义

APSS 不应一开始设计成复杂 agent，而应是一个结构选择器：

1. 从简单模型开始；
2. 逐步考虑非单调边际函数和交互项；
3. 每次结构扩展都必须通过 validation gate；
4. 若复杂结构带来的提升低于阈值，则保留更简单结构；
5. 最终输出预测性能和被选择的结构。

一个可实现的选择准则是：

```text
selected_model = argmin(validation_loss + λ * structure_complexity)
```

其中 `structure_complexity` 可以包括：

- 非单调准则数量；
- 交互项数量；
- 边际函数自由度；
- 总参数量；
- 是否使用 full flexible module。

### 3.3 最小充分结构原则

文章的核心不应是“模型越复杂越好”，而应是：

> 能用简单结构解释，就不引入复杂结构；只有当验证证据足够强时，才允许非单调或交互结构进入模型。

这比“自动发现所有东西”更稳，也更容易被审稿人接受。

## 4. 数据与 benchmark 设计

当前已经准备好的数据见：

- `data/metadata/bench_manifest.csv`
- `data/metadata/bench_quality_report.md`
- `data/processed/`

实验应分三层，而不是只跑真实数据 accuracy。

### 4.1 Layer A：Synthetic structure-recovery benchmark

目的：验证模型是否真的能发现结构，而不是只会分类。

已准备数据：

| dataset_id | 真值结构 | 主要检验 |
|---|---|---|
| synthetic_monotone_additive | 所有准则线性单调、无交互 | 是否避免虚假复杂化 |
| synthetic_nonmonotone_marginals | g1 倒 U 型，g2 U 型，其余单调 | 是否发现非单调准则 |
| synthetic_sparse_interactions | g1-g2 正交互，g3-g4 负交互 | 是否发现稀疏交互 |
| synthetic_mixed_structure | g1 非单调，g2-g5 正交互，g3-g6 负交互 | 是否能组合选择结构 |

第一版可直接用这 4 个数据集。第二版应扩展成多 seed、多噪声、多样本规模：

- sample size：200 / 600 / 1200；
- noise level：0 / 0.025 / 0.05 / 0.10；
- random seed：至少 20 次；
- class number：固定 4 类，后续可扩展。

### 4.2 Layer B：Public monotone classification benchmark

目的：检验模型在已知 gain-type ordered criteria 的任务上，是否会错误引入非单调或交互结构。

已准备数据：

- mcs_bcc
- mcs_cev
- mcs_cpu
- mcs_dbs
- mcs_era
- mcs_esl
- mcs_lev
- mcs_mmg
- mcs_mpg

这些数据的 README 明确说明：

- 所有 criteria 为 gain type；
- criteria 已归一化到 [0, 1]；
- class index 越大越优。

预期：如果 APSS 是合理的，它不应在这类数据上大面积选择非单调结构。

### 4.3 Layer C：Application benchmark

目的：检验框架在接近真实 MCDA 的任务中是否有用。

#### C1. Polish research units

用途：主应用场景。它有官方机构评价标签，比 QS 更像真实 MCDA 应用。

数据：

- polish_research_units_all
- polish_research_units_hs
- polish_research_units_nz
- polish_research_units_si
- polish_research_units_ta
- polish_research_units_njn

预期：不提前假设一定有交互。我们观察 APSS 是否稳定选择某些交互或保持简单。

#### C2. QS university rankings

用途：外部排名监督信号，不作为客观真值。

数据：

- qs_2020_top500_common6
- qs_2022_top500_common6
- qs_2026_top500_common6
- qs_2026_top500_all10

实验设计：

- within-year repeated split；
- cross-year generalization：2020/2022 训练或验证，2026 测试；
- common6 用于跨年一致比较；
- all10 仅作为 2026 扩展实验。

预期：QS 可能更接近加权公式，因此 APSS 若选择简单结构反而是好结果。这能证明框架不是一味复杂化。

#### C3. Wine quality

用途：非线性/非单调可能性较高的公开有序分级任务。

数据：

- wine_quality_red
- wine_quality_white
- wine_quality_combined

预期：相比 monotone additive，APSS 可能更倾向选择非单调或非线性边际结构。但由于质量标签极不均衡，必须重点看 macro-F1、balanced accuracy 和 ordinal MAE，而不是只看 accuracy。

## 5. 实验拆分策略

### 5.1 第一阶段：快速可行性实验

用于判断方向是否值得继续。

- 每个数据集 repeated stratified split；
- train / validation / test = 60 / 20 / 20；
- seeds = 20；
- validation 用于结构选择；
- test 只用于最终评估。

输出：

- 每个模型在每个数据集上的平均性能；
- APSS 选择结构的频率；
- synthetic 上的结构恢复指标；
- 是否存在明显的“不同数据需要不同结构”证据。

### 5.2 第二阶段：正式实验

用于论文主结果。

- nested cross-validation；
- outer folds 用于最终测试；
- inner folds 用于模型选择和超参数选择；
- 每个数据集重复多次；
- 小样本数据避免过重模型；
- 所有随机种子固定并记录。

推荐正式设置：

- synthetic：30 seeds × 多噪声 × 多样本规模；
- real benchmark：5 outer folds × 3 repeats；
- 如果计算压力过大，先对 9 个 monotone datasets 和 Polish all/subsets 做完整实验，再扩展 QS/wine。

## 6. 指标设计

### 6.1 预测性能

由于任务是有序类别，不能只看普通 accuracy。

必须报告：

- accuracy；
- macro-F1；
- balanced accuracy；
- ordinal MAE：`mean(abs(y_pred - y_true))`；
- Spearman 或 Kendall correlation；
- top-1 accuracy / within-one-class accuracy。

其中：

- accuracy 便于和 MCDA sorting 文献对比；
- macro-F1 处理类别不均衡；
- ordinal MAE 体现有序类别错误大小；
- Kendall/Spearman 体现排序一致性。

### 6.2 结构恢复指标

只适用于 synthetic benchmark。

必须报告：

- monotone criterion detection accuracy；
- non-monotone criterion precision / recall / F1；
- interaction pair precision / recall / F1；
- false complexity rate；
- selected model family accuracy；
- threshold recovery error；
- marginal function shape error。

### 6.3 复杂度与稳定性

用于证明“最小充分结构”。

必须报告：

- selected non-monotone criteria count；
- selected interaction count；
- effective degrees of freedom；
- parameter count；
- complexity penalty value；
- selected structure stability across folds；
- selected interaction Jaccard similarity；
- marginal shape stability。

### 6.4 计算成本

用于避免审稿人质疑“只是暴力搜索”。

报告：

- training time；
- model selection time；
- number of candidate structures evaluated；
- peak memory，如可获得；
- failed runs 或 non-convergence count。

## 7. 基线设计

基线必须分层，不然会被质疑不公平。

### 7.1 Sanity baselines

| baseline | 作用 |
|---|---|
| Majority class | 检查任务难度下限 |
| Stratified random | 检查类别分布影响 |

### 7.2 传统可解释 MCDA / preference learning baselines

| baseline | 对应文献或思想 | 作用 |
|---|---|---|
| Linear additive + thresholds | 最简单 value sorting | 检验简单加性结构是否足够 |
| Monotone additive value model | UTADIS / monotone MCDA sorting 思想 | 传统单调结构基线 |
| Non-monotone additive value model | Liu et al. 2019 | 检验非单调边际函数价值 |
| Diverse additive marginal models | Liu et al. 2020 | 检验不同边际函数族 |
| Sparse interaction model | Liu et al. 2020 INFORMS JOC | 检验准则交互价值 |
| Choquet-style interaction baseline | Choquet sorting 思想 | 作为交互建模对照，若实现成本可控 |

第一阶段不必精确复现所有论文模型，但至少要实现其结构等价或近似版本。正式论文阶段应优先复现：

1. monotone additive；
2. non-monotone additive；
3. interaction-based model；
4. APSS selector。

### 7.3 参数学习 baselines

这一组用于回应录音中的原始问题：传统方法中的人工参数能否由数据自动学习。

| baseline | 对应原始问题 | 作用 |
|---|---|---|
| Grid / random search tuning | 对应智能 ELECTRE III 中的暴力搜索式参数寻找 | 检验 APSS 是否优于简单搜索 |
| Bayesian optimization tuning | 更标准的黑箱参数优化 | 检验是否只是普通参数优化问题 |
| Neural parameter predictor | 对应用 NN 学 q/p/v 的团队样板 | 检验“固定结构参数学习”的强度 |
| RL / bandit-style parameter search | 对应录音中的奖励惩罚机制 | 作为后续扩展或强对照 |

第一阶段可以先实现 grid/random search 与 neural-style predictor 的简化版本。RL / bandit 可以先作为扩展路线，不必一开始实现。

这组 baseline 的核心问题是：

> 如果只学习固定结构里的参数，是否已经足够？APSS 学习更高层结构是否真的有额外价值？

### 7.4 普通机器学习 baselines

这些不是主贡献，但必须有，否则会被 ML 审稿人质疑。

| baseline | 作用 |
|---|---|
| Multinomial / ordinal logistic regression | 简单 ML 基线 |
| SVM | 传统分类强基线 |
| Random forest | 非线性非参数基线 |
| Gradient boosting / HistGradientBoosting | 强 tabular baseline |
| MLP | black-box neural baseline |

这些模型主要用于回答：

> APSS 是否只是一个性能较弱的可解释模型？还是能在可解释性和性能之间取得可接受平衡？

### 7.5 结构选择对照 baselines

这一组很关键，因为我们的贡献是“结构选择”。

| baseline | 作用 |
|---|---|
| Always-simple | 永远选择最简单模型 |
| Always-full | 永远选择最复杂模型 |
| Best-validation without complexity penalty | 检验复杂度惩罚是否必要 |
| APSS without interaction module | 检验交互模块贡献 |
| APSS without non-monotone module | 检验非单调模块贡献 |
| Black-box model selector | 检验 APSS 是否只是普通 AutoML |

这组实验决定文章能不能成立。因为我们真正要证明的是：

> APSS 不是“更复杂模型”，而是“知道什么时候该复杂、什么时候不该复杂”。

## 8. 预期结果长什么样

### 8.1 支持我们故事的结果

如果实验支持该方向，应看到以下模式：

1. 在 synthetic_monotone_additive 上：
   - APSS 大多数时候选择 M1/M2；
   - 非单调和交互 false positive 很低；
   - 性能接近 always-full，但复杂度更低。

2. 在 synthetic_nonmonotone_marginals 上：
   - APSS 能识别 g1/g2 的非单调；
   - M3/M5 明显优于 M1/M2；
   - APSS 接近 oracle best。

3. 在 synthetic_sparse_interactions 上：
   - APSS 能选择 g1-g2、g3-g4 这类交互；
   - interaction F1 高于随机或 always-full 的稀疏解释；
   - sparse interaction 优于纯 additive。

4. 在 monotone public benchmark 上：
   - APSS 多数时候保持简单或单调；
   - 与非单调/全模型性能接近；
   - 复杂度显著低于 always-full。

5. 在 wine quality 上：
   - APSS 选择更多非单调/非线性结构；
   - macro-F1、ordinal MAE 优于单调模型；
   - 如果 black-box 更强，APSS 至少应提供接近性能和更好解释。

6. 在 QS 上：
   - APSS 很可能选择简单加性或少量非线性；
   - 这不是失败，而是支持“框架不会盲目复杂化”；
   - cross-year 结果比 within-year 更重要。

### 8.2 推翻我们故事的结果

以下结果会削弱甚至推翻该方向：

1. Synthetic 上 APSS 无法恢复真值结构；
2. Always-full 在所有数据上都显著优于 APSS，且没有明显过拟合；
3. Always-simple 与 APSS 几乎无差异，说明结构选择没有必要；
4. APSS 选择结构在不同 folds/seeds 中极不稳定；
5. APSS 相比普通 ML 没有性能优势，也没有足够解释优势；
6. 复杂度惩罚只是人为调参，无法稳定迁移到不同数据集；
7. 真实数据上的结构解释全是 post-hoc，不具备复现性。

这部分必须提前写清楚，否则后面容易只讲好结果。

## 10. 统计分析策略

第一阶段：

- 以描述性统计为主；
- 报告 mean ± std；
- 报告每个数据集的结果，不只报告平均值；
- 不急着做强统计声明。

正式阶段：

- 对 repeated splits 使用 paired comparison；
- 多模型多数据集比较可使用 Friedman test + Nemenyi / Holm correction；
- 对 APSS vs key baseline 使用 Wilcoxon signed-rank；
- 报告 effect size，而不是只报 p-value；
- 对 synthetic 结构恢复报告置信区间。

注意：

- 数据集数量有限时，不应过度解释显著性；
- repeated split 不等于独立样本，统计检验要保守；
- QS 不能用于“真实偏好”结论，只能用于外部排名复现/泛化。

## 11. 最小可执行路线

### Step 1：先跑可行性 baseline

实现：

- M0 majority；
- M1 linear additive + thresholds；
- M3 non-monotone additive；
- M4 sparse interaction；
- M5 full flexible；
- M6 APSS selector。

先不追求完全复现 UTADIS / Liu et al. 的优化模型，而是用结构等价的 thresholded score model 做 proof-of-concept。

### Step 2：验证 synthetic

先看 APSS 是否能通过最基本的结构恢复测试。

如果 synthetic 都过不了，不进入真实数据。

### Step 3：跑 monotone suite

检验 APSS 是否会过度复杂化。

如果在 monotone suite 上大量选择非单调和交互，说明 validation gate 或 complexity penalty 有问题。

### Step 4：跑 Polish / QS / Wine

观察真实任务中的结构差异。

重点不是“哪个数据最高”，而是：

- 哪些数据需要简单结构；
- 哪些数据需要非单调；
- 哪些数据需要交互；
- 这些选择是否稳定；
- 对应的案例解释是否可信。

### Step 5：补强文献级 baseline

如果 proof-of-concept 成立，再补：

- UTADIS-like baseline；
- Liu et al. 2019 non-monotone regularized additive baseline；
- Liu et al. 2020 interaction baseline；
- MLP / deep preference learning baseline；
- possibly Choquet baseline。

## 12. 当前最重要的判断

这项研究是否值得继续，不取决于我们能否马上做出最强模型，而取决于第一个 evidence packet 是否显示：

> 不同多准则 benchmark 确实偏好不同结构，而验证门控的 APSS 能选择接近最优但更紧凑的结构。

如果这个证据成立，文章故事会比较稳：

> 现有方法通常先选定一个偏好结构族，再学习参数；我们研究的是偏好结构本身如何在验证证据下被选择，并证明“何时复杂、何时简单”比单纯提出更复杂模型更重要。

如果这个证据不成立，应及时收缩或放弃该方向。

## 14. 主要文献边界

- Liu et al. (2019)：非单调准则 preference disaggregation，不是我们的空白。
- Liu et al. (2020, EJOR)：valued assignment + diverse additive value models，不是我们的空白。
- Liu et al. (2020, INFORMS JOC)：interacting criteria，不是我们的空白。
- EJOR 2023 deep preference learning：深度学习 MCDA，不是我们的空白。
- Omega 2025：large-scale non-monotonic ML sorting，需要避免正面重复。
- Information Fusion 2026：attention interaction sorting，需要避免“attention 学交互”式撞车。

我们的边界应是：

> 不是提出某一种结构，而是在多个候选可解释结构之间进行验证门控选择，并通过 synthetic + public benchmark + application benchmark 证明结构选择本身的价值。
