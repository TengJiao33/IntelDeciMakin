# 原始问题与 APSS 实验设计对齐审查

日期：2026-06-29  
依据：`docs/新录音-音频_原文 (6).docx`  
方法：research-story-synthesis 成熟度审查

## 1. Readiness

solid_candidate，但不是 novel_candidate。

当前 APSS 方案与老师最初提出的问题有明显呼应，但仍需要校正叙事边界。它不能被写成一个泛化到所有决策问题的自演化框架，而应被写成：

> 在智能多属性决策中，将传统方法里需要决策者人工给定的参数和结构假设，转化为可由数据训练、验证门控、复杂度约束来学习和选择的对象。

## 2. 录音中最初的问题是什么

录音中老师给出的原始问题不是“做 benchmark”本身，也不是“做一个通用 AI 框架”。它的核心是：

> 传统多属性决策方法存在大量需要决策者人工设定的参数、函数或偏好信息；在智能时代，应利用前沿 AI 技术，使这些人工设定能够通过数据训练自动学习，从而实现智能多属性决策。

可以拆成四层。

### 2.1 背景判断：智能多属性决策体系尚不成熟

老师首先强调，智能多属性决策方向还没有形成成熟体系。相比在线评论多属性决策已经研究多年，智能多属性决策文献较少，体系也不完整。因此学生需要在已有文献启发下产生自己的想法。

这意味着：

- 该方向允许探索；
- 但更需要清楚的问题定义和验证方式；
- 不能只是套一个 AI 名词。

### 2.2 方法线：改造传统 MADM 方法

老师明确说有“两条思路”，第一条是“纯方法”，即不做具体应用，而是针对传统理论/方法中的缺陷进行智能化改进。

传统方法的主要缺陷被表述为：

- 前景理论、后悔理论、TODIM 等方法中有风险偏好、损失规避等参数；
- ELECTRE III 中有 indifference/preference/veto thresholds，即 q/p/v；
- PROMETHEE 等方法中有偏好函数及其参数；
- 这些参数往往需要决策者自己给定；
- 许多决策者并不知道这些参数的深层含义，也不知道怎么设。

因此，老师最初的问题可以概括为：

> 能否利用 AI 技术，让传统 MADM 方法中的人工参数和人工选择自动学习出来，从而减少或取消决策者参与？

### 2.3 应用线：在线评论智能排序

第二条思路是应用线，尤其是在线评论多属性决策。老师认为在线评论中，文本处理已经大量使用机器学习和深度学习，但“产品排序/决策层”仍多用传统 MADM 方法，没有充分智能化。

这条线的关键问题是：

- 在线评论可以作为数据驱动智能决策的场景；
- 产品排序需要一个基准或奖励信号；
- 可能用评论数量、平台排序、销量等作为弱 benchmark；
- 如果用强化学习，也可能通过奖励惩罚机制探索，不一定依赖固定基准。

但是老师也承认这条线需要摸索，尤其是 benchmark 和数据问题。

### 2.4 验证线：实验必须有明确指标

录音中反复强调实验验证。老师用已有智能 ELECTRE III 稿件和用户前两篇论文经验来说明：

- 想法可以新奇，但必须被实验验证；
- 最好有明确指标，如 accuracy、precision、F1 等；
- 实验优势越明显，论文说服力越强；
- 第二篇论文的问题之一就是实验验证不够强；
- 大学排名数据可用于方法线验证，且每年更新，适合做泛化实验。

所以，最初问题里其实包含一个很强的实验要求：

> 智能化方法不能只讲概念，必须通过有明确指标的数据实验证明优于传统方法或能解决传统方法的缺陷。

## 3. 老师给出的样板：智能 ELECTRE III

录音中老师重点讲了团队已有的智能 ELECTRE III 论文，其逻辑是：

1. 传统 ELECTRE III 需要决策者给定 q/p/v；
2. q/p/v 含义复杂，决策者难以设定；
3. 用神经网络从数据中学习 q/p/v；
4. 输入问题数据后，模型自动给出最优参数；
5. 用大学排名数据验证排序效果；
6. 用 2026 年数据做泛化实验。

这个样板给我们三个启发：

- 原始问题是“人工参数智能化”；
- 数据验证是核心；
- 大学排名数据是老师认可的可用 benchmark。

但老师也指出这篇论文的问题：

- 实质上有暴力搜索成分；
- 运行代价不低；
- 想法清楚，但技术实现不一定足够高级。

这为我们提供了一个可以上移的问题：

> 如果只学习某个固定方法的几个参数，例如 ELECTRE III 的 q/p/v，是否仍然太局部？能不能进一步学习更一般的偏好结构？

## 4. 现在 APSS 实验设计与原始问题的呼应

### 4.1 呼应上的部分

APSS 与原始问题的核心呼应在于：

| 录音中的原始问题 | APSS 中的对应设计 |
|---|---|
| 传统方法需要决策者设定参数 | APSS 将参数、函数形状、交互项、阈值都视为可学习对象 |
| 决策者不知道 q/p/v、风险偏好等怎么设 | APSS 通过 validation gate 从数据中选择结构和参数 |
| 智能决策要靠数据训练 | APSS 使用 synthetic + public benchmark + application benchmark |
| 实验要有明确指标 | APSS 设计 accuracy、macro-F1、ordinal MAE、结构恢复、复杂度等指标 |
| 大学排名数据可用于方法验证 | APSS 纳入 QS 2020/2022/2026 |
| 需要泛化实验 | APSS 设计 cross-year QS generalization 和多 benchmark 验证 |
| 不要只靠概念 | APSS 先要求 synthetic structure recovery，通过后才进入真实数据 |

所以，APSS 没有偏离原始问题。它是在原始“参数智能化”的基础上，把对象扩展为“偏好结构智能化”。

### 4.2 上移了一层的部分

原始问题最直接的版本是：

> 用 AI 学传统方法中的参数。

APSS 的版本是：

> 用数据和验证门控选择传统方法背后的偏好结构，包括参数、单调性、边际函数、交互项和阈值。

这是一种上移：

- 从学习 q/p/v 这种具体参数；
- 上移到学习“是否应该使用某类偏好结构”；
- 从固定 ELECTRE III；
- 上移到多种可解释 sorting model 的结构选择。

这种上移是有创新潜力的，但也带来风险：

- 容易显得不像传统 MADM，而像 AutoML；
- 容易偏离老师说的“前沿 AI 技术改进传统方法”；
- 如果不绑定具体 MADM 结构，会变成普通分类模型选择；
- 如果不纳入传统方法基线，会削弱与原始任务的联系。

## 5. 当前实验设计需要校正的地方

### 5.1 需要把“人工参数”重新放回故事中心

现在实验设计里强调了结构选择，但原始问题首先是人工参数问题。因此 APSS 叙事需要改成：

> 传统 MADM 的人工设定不仅包括数值参数，也包括偏好函数、单调性假设、交互假设和类别阈值。已有研究多集中于学习某一种固定结构中的参数，而本文尝试学习更高层的偏好结构。

这样才能从老师的 q/p/v 样板自然过渡到 APSS。

### 5.2 需要加入“固定参数学习”的基线

实验设计里目前有 monotone/non-monotone/interaction baselines，但还需要补一类基线：

| baseline | 作用 |
|---|---|
| Grid / random search parameter tuning | 对应智能 ELECTRE III 中的暴力搜索问题 |
| Bayesian optimization parameter tuning | 更标准的黑箱参数优化基线 |
| Neural parameter predictor | 对应团队已有 NN 学 q/p/v 的思想 |
| RL / bandit-style parameter search | 对应录音中强化学习奖励惩罚思想 |

这些 baseline 不一定第一阶段全部实现，但需要在设计中占位。否则 APSS 与录音中的“参数智能化”联系会弱。

### 5.3 需要明确 APSS 不是替代所有传统 MADM 方法

APSS 应先限定在：

> 有标签多准则 sorting / ranking-to-sorting 任务。

不要说它适用于所有 MADM。因为录音里的问题虽然宏大，但老师具体讲的验证样板仍然是大学排名、ELECTRE III、参数学习和排序准确率。

### 5.4 需要保留“前沿 AI 技术”入口，但不要一开始就做大

录音中老师提到深度学习、强化学习、奖励惩罚机制。APSS 第一阶段可以不直接使用 RL，但需要解释：

- validation gate 相当于一种保守的奖励/惩罚机制；
- 结构扩展只有带来验证收益才被接受；
- 后续可以将结构搜索器替换为 bandit / reinforcement learning / Bayesian optimization。

第一阶段应写成：

> 先用可解释、可控的结构选择验证问题是否成立；如果成立，再考虑更复杂的 RL-style search。

## 6. 是否呼应上了：结论

结论：呼应上了，但需要重新锚定。

更准确地说：

> APSS 不是脱离录音原始问题的新方向，而是把“传统 MADM 参数智能化”推进到“传统 MADM 偏好结构智能化”。它呼应了老师关于人工参数、数据训练、AI 改进传统方法、大学排名验证和明确实验指标的要求。但它必须继续绑定可解释 MADM 结构和传统方法基线，否则会漂移成普通机器学习模型选择。

因此，当前实验设计应被重新解释为：

> 原始问题：传统 MADM 方法依赖决策者人工给定参数和结构，智能化不足。  
> 我们的推进：不仅学习参数，还学习何种偏好结构是必要的。  
> 实验证据：用 synthetic 检验结构恢复，用 public benchmark 检验泛化，用 QS/Polish/Wine 检验真实场景表现。  
> 最小目标：证明模型知道什么时候该保持简单，什么时候需要非单调或交互。

## 7. 对原实验设计的修改建议

应对 `docs/自适应偏好结构学习_实验设计方案_2026-06-29.md` 做三处补强：

1. 在开头加入“与录音原始问题的对应关系”。
2. 在 baseline 章节补充 parameter-learning baselines。
3. 在方法章节明确 APSS 是“参数智能化 → 结构智能化”的扩展，而不是普通 AutoML。

## 8. 下一步最小证据对象

在正式跑实验前，应先实现一个最小 proof-of-concept，用来回答：

1. 固定结构参数学习是否能解决一部分问题？
2. APSS 是否比固定结构参数学习更稳？
3. synthetic 上是否能恢复已知结构？
4. monotone benchmark 上是否会避免错误复杂化？

如果这四个问题没有正面证据，APSS 不应继续扩展。
