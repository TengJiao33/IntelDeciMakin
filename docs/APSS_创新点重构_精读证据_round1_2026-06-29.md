## The uncomfortable but useful conclusion

Our first version of the motivation was too broad. It is not accurate to say that existing decision methods simply use preset formulas and do not learn. The close literature already learns:

- MRSort / ELECTRE-like parameters from assignment examples;
- criteria used by the DM under additive preference disaggregation;
- nonmonotone criterion types such as gain, cost, single-peaked, and single-valley;
- sparse interaction patterns in capacity-based aggregation models.

The real gap is narrower and sharper:

> Existing work usually learns inside a pre-selected interpretable preference model. APSS should learn one layer up: which interpretable preference structure is warranted by the data, and when added structure is false complexity.

## Revised problem statement

Instead of asking:

> Can AI make MADM more intelligent?

We should ask:

> Given a decision dataset, can an adaptive decision framework determine whether the underlying preference explanation should be additive, criteria-sparse, nonmonotone, interaction-aware, or still simple, while preserving predictive performance and avoiding spurious structures?

This directly responds to the original benchmark concern: if the output ranking/sorting has no external truth, we need settings where at least some labels or structure-truth signals exist.

## Innovation candidates that survive the literature

### 1. Adaptive preference-structure governance

Core idea: not "learn parameters," but select among interpretable preference-structure hypotheses.

Candidate structure families:

- monotone additive;
- criteria-sparse additive;
- nonmonotone / single-peaked criterion forms;
- sparse interaction forms;
- fixed MRSort-like majority-rule sorting;
- black-box predictive upper bound, used only as a reference.

Why this survives:

- IJCAI 2023 learns sparse interactions but inside capacity-based aggregation.
- PDA 2025 selects criteria but inside additive value functions.
- MRSort nonmonotone learns criterion types but inside MRSort.
- Sobrie 2019 learns MRSort parameters but keeps monotone MRSort structure.

So APSS can be framed as an adaptive controller over interpretable decision structures, not a single new preference model.

### 2. False-complexity control as decision-explanation error

Core idea: a discovered interaction or nonmonotone criterion is not automatically good. If it is unnecessary or unstable, it is a false explanation.

This is stronger than ordinary regularization:

- Ordinary regularization says complex models may overfit.
- APSS can say spurious preference structures are harmful because they misexplain the decision logic.

Why this survives:

- PDA 2025 controls complexity, but within additive criteria selection.
- IJCAI 2023 promotes sparse interactions, but does not make "do we need interaction at all?" the central governance problem.
- MRSortNonMonotone2021 shows high assignment restoration can coexist with wrong criterion-type recovery in the ASA case. This is exactly the phenomenon APSS should formalize.

Required metric:

- predictive: accuracy, macro-F1, ordinal MAE;
- complexity: selected criteria count, interaction count, nonmonotone count;
- false complexity: spurious selected criteria / interactions / nonmonotone types under synthetic structure truth;
- stability: structure persistence across splits and perturbations.

### 3. Dual benchmark: downstream performance plus structure truth

Core idea: APSS is not evaluated only by "does it predict labels." It must also answer whether the learned decision explanation is structurally faithful.

This is not strong enough alone as a method innovation, but it is necessary to support the first two innovations.

Benchmark layers:

- synthetic controlled tasks: known monotonicity, criteria relevance, and interaction truth;
- public ordinal classification / sorting tasks: external labels for predictive comparison;
- downstream decision simulation: top-k selection, category assignment, or ranking-to-action cost.

## What we should not claim

- "First to learn interactions."
- "First to select criteria."
- "First to learn nonmonotone preferences."
- "First to use regularization in preference disaggregation."
- "First to learn MCDA model parameters from examples."

## Baseline obligations

The literature implies that our baselines cannot be only mean/TOPSIS/VIKOR/TODIM.

Minimum baseline families:

- additive monotone learner;
- criteria-selection additive learner;
- nonmonotone criterion-type learner or practical approximation;
- sparse interaction learner;
- MRSort / UTADIS-style fixed-family learner;
- black-box ML upper bound such as random forest or gradient boosting;
- APSS ablations: no gate, top-k interactions, forward-gain interactions, stability-gated interactions.

## Current risk

If APSS only becomes "add a parameter-learning module to a decision method," the novelty collapses. PDA 2025 and MRSort learning already cover that pattern.

If APSS only becomes "learn interactions," IJCAI 2023 is stronger.

If APSS only becomes "learn nonmonotone criteria," MRSortNonMonotone2021 is directly there.

Therefore, the paper must be built around cross-structure adaptivity and false-complexity control.

## Decision-journal positioning adjustment

Important correction: our target is a decision / MCDA journal, not necessarily an AI or CS top conference. Therefore, "collision" should not be interpreted as "this cannot be done." A more useful interpretation is:

> If a technical component already exists in AI/preference-learning literature, APSS can still use it, but the paper must clearly reposition the contribution at the decision-research level: decision problem formulation, interpretable preference-structure governance, validation under decision tasks, and practical decision-support value.

This means some overlap is acceptable and even useful:

- IJCAI-style sparse interaction learning can be treated as evidence that interaction discovery is technically feasible, while APSS asks when a decision analyst should trust or reject interaction structure.
- PDA-style criteria selection can be treated as a close MCDA component, while APSS extends the question from "which criteria inside an additive model" to "which preference structure should be used for this decision task."
- MRSort nonmonotone learning can be treated as a specialized structure learner, while APSS asks whether nonmonotonicity is necessary, stable, and decision-relevant.

The framing should soften from "we are first" to "we integrate and govern":

- Not: "Existing methods cannot learn preferences."
- Better: "Existing methods typically learn within a chosen preference model; decision analysts still lack a systematic way to decide which interpretable preference structure is justified by data."
- Not: "We invent interaction discovery."
- Better: "We treat interaction discovery as one candidate structure and control against spurious interactions."
- Not: "We beat all existing MCDA models."
- Better: "We improve adaptive decision-model selection while preserving interpretability and reducing false structural explanations."

For a decision journal, the contribution can be packaged as a decision-support framework rather than a pure algorithmic novelty claim. The expected reviewer question is not only "is the optimizer new?", but "does this help decision analysts build more credible, data-supported, interpretable decision models?"

## Two peer groups and revised contribution order

We should separate two audiences in the literature review.

### Peer group A: decision / MCDA journal peers

This is the primary differentiation field. These papers care about:

- decision problem formulation;
- preference elicitation and preference disaggregation;
- interpretability and analyst/DM interaction;
- sorting/ranking/choice validity;
- robustness, sensitivity, and explainability;
- whether the method helps decision analysts reach more credible recommendations.

Against this group, APSS should be positioned as a decision-support contribution:

> Existing MCDA learning methods usually ask the analyst to choose a preference model first, then infer parameters or a limited structural component. APSS moves the analyst's burden upward: it adaptively compares plausible interpretable preference structures and recommends the simplest structure that is sufficiently supported by data.

Here the novelty does not need to be "we invented every learner." It can be:

- a new decision-modeling workflow;
- a new adaptive preference-structure selection problem;
- a practical validation protocol for credible decision explanations;
- a framework that integrates multiple interpretable MCDA structures under one governance logic.

### Peer group B: AI / CS conference peers

This is not the primary battlefield, but it defines technical collision boundaries. These papers show that:

- sparse interaction learning is technically possible;
- feature/criteria selection and regularization are mature;
- AutoML/model selection exists;
- scalable optimization for preference models exists.

Against this group, APSS should not overclaim algorithmic firstness. Instead, these papers become enabling/adjacent evidence:

> We borrow or approximate relevant learning components, but use them inside a decision-analysis framework whose goal is not generic prediction, but interpretable preference-structure governance for decision tasks.

### Revised innovation arrangement

The contribution order should be changed from "algorithm first" to "decision framework first."

#### Contribution 1: A new decision-analysis problem formulation

Proposed wording:

> We formulate adaptive preference-structure selection as a decision-support problem: given assignment/preference evidence, the system should not only estimate model parameters, but also decide whether additive, sparse-criteria, nonmonotone, or interaction-aware preference structures are warranted.

Why this should be first:

- It speaks directly to decision journals.
- It makes APSS more than a collection of ML tricks.
- It explains why online-review MADM felt weak: without labels or structure truth, method validity becomes hard to defend.

#### Contribution 2: An interpretable APSS framework for structure governance

Proposed wording:

> We develop an interpretable APSS framework that organizes multiple MCDA-compatible learners under a common governance mechanism, balancing predictive restoration, structural simplicity, and stability.

This is the main method contribution, but it is framed as a framework for decision analysts, not as a brand-new optimizer.

Components can include:

- monotone additive learner;
- criteria-sparse learner;
- nonmonotone criterion learner;
- sparse interaction learner;
- structure gate / stability gate / complexity gate.

#### Contribution 3: False-complexity-aware validation

Proposed wording:

> We introduce false-complexity diagnostics to distinguish useful preference structures from spurious ones, showing that high assignment accuracy can coexist with misleading recovered structures.

This becomes stronger if paired with the MRSortNonMonotone2021 observation: high restoration can still recover wrong criterion type unless data are informative.

This contribution should not be presented as a standalone benchmark novelty. It should support Contribution 2.

#### Optional Contribution 4: Multi-scenario empirical demonstration

This should be written as empirical value, not core novelty:

> We demonstrate the framework across controlled structure-truth benchmarks and public ordinal decision datasets, showing when APSS selects simpler structures and when it justifiably activates nonmonotone or interaction-aware modules.

This gives the paper credibility, but should not be the headline innovation.

## New abstract-level story

The paper can now be narrated as:

> In MCDA, learning from assignment or preference examples has become increasingly common, but most methods still require analysts to commit to a preference model family before learning begins. This creates a practical dilemma: overly simple models may miss nonmonotonicity or interactions, whereas overly flexible models may produce persuasive but spurious preference explanations. We formulate adaptive preference-structure selection as a decision-support problem and propose APSS, an interpretable framework that governs multiple MCDA-compatible structure learners under predictive, complexity, and stability criteria. Experiments on controlled structure-truth tasks and public ordinal decision datasets examine whether APSS can preserve decision performance while reducing false structural explanations.

This framing lets us absorb partial technical overlap with AI/CS work while differentiating in the decision-journal space.
