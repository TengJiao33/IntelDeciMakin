# 同行创新点写法样本

Date: 2026-06-29

目的：抽取附近 MCDA / MCDM / preference learning 文献中“创新点如何写”的真实组织方式。以下多为近义概括，不逐字摘抄，避免把原文当模板机械复制。

## 观察结论

同行的创新点通常不写成“我们提出一个观点”。他们更常写成：

- 提出一个 named method / DSS / model / algorithm；
- 引入一个 new index / descriptor / variable / measure；
- 扩展一个 known model 到一个明确新情形；
- 给出一个 optimization formulation；
- 给出一个 procedure / strategy / query rule；
- 用 benchmark / case study 证明方法对象有效。

## 写法模式提炼

从上面 50 个样本看，同行更喜欢以下句型：

1. We propose/develop a [named method] for [specific task].
2. We introduce a [new index/vector/variable] to measure/represent [specific object].
3. We extend [known model] to [new condition].
4. We formulate [task] as [MIP/LP/optimization/problem].
5. We design [algorithm/procedure/strategy] to handle [specific bottleneck].
6. We compare [proposed object] with [named baselines] on [named benchmarks].
7. We demonstrate [capability] through [synthetic experiment / real case / benchmark].
8. We provide [taxonomy/database/software] that supports [analyst task].

## 对我们 APSS 的直接启发

不能写：

- We formulate preference-structure uncertainty as an important issue.
- We propose an adaptive decision framework.
- We consider both accuracy and interpretability.

更像同行的写法应该是：

- We propose a structure-gated preference disaggregation method for jointly learning criteria relevance, monotonicity type, and interaction activation from assignment examples.
- We introduce a Preference-Structure Acceptability Index to quantify the empirical support of competing interpretable preference structures.
- We define structure-necessary and structure-possible recommendations to assess whether a decision result is robust across supported preference structures.
- We develop an iterative structure activation algorithm that starts from an additive baseline and activates nonmonotone or interaction terms only when they improve fit under stability and parsimony constraints.

换句话说，创新点要从“主张”改成“对象”。
