# 自适应偏好结构学习框架：研究构想记录

日期：2026-06-29

## 1. 当前核心判断

我们不应把新研究线表述为泛泛的“AI 赋能 MADM”或“用神经网络改进多准则决策”。这类表述已经不够新，也容易与现有深度偏好学习、非单调偏好学习、交互准则学习、attention-based sorting 等研究撞车。

更稳、更有研究价值的表述是：

> 传统 MADM/MCDA 方法中大量依赖人工预设的建模选择，例如准则单调性、边际价值函数形状、准则交互关系、类别阈值、模型复杂度等，可以被重新定义为一个由数据驱动、验证门控、正则约束的偏好结构学习问题。

因此，我们可能的核心问题不是“提出另一个固定 MADM 模型”，而是：

> 如何构建一个能够自动发现、选择并验证偏好结构的通用多准则排序/分级框架？

## 2. 与自演化 Agent / SkillOpt 思想的关系

近期自演化 agent 与 SkillOpt 一类思想的关键启发，不在于“模型神奇地产生智能”，而在于：

- 将外部结构视为可优化对象；
- 用任务 rollout 或 benchmark 表现产生证据；
- 对结构修改设置验证门控；
- 保留失败修改作为负经验；
- 通过受限、可回滚、可审计的更新逐步改进系统。

这一思想可以迁移到 MADM/MCDA，但不应直接迁移 agent 外壳，而应迁移其“外部结构可学习”的机制。

对应关系如下：

| 自演化 Agent / SkillOpt | 多准则决策中的对应物 |
|---|---|
| skill document 是可训练外部状态 | 偏好结构是可训练外部状态 |
| 人工规则被数据驱动编辑替代 | 人工预设单调性、权重、阈值、交互项被数据驱动选择替代 |
| rollout 产生成功/失败证据 | benchmark 上的分类、排序、稳定性表现产生证据 |
| validation gate 决定是否接受 skill edit | held-out accuracy、F1、Kendall tau、calibration 等决定是否接受结构变化 |
| rejected edits 形成失败记忆 | 被验证失败的偏好结构进入禁用或惩罚集合 |
| 输出 compact skill | 输出 compact interpretable decision model |

## 3. 适合迁移的前提

该思想适合迁移到多准则决策问题中，但有一个硬前提：

> 必须存在外部评价信号。

如果研究对象仍然是纯 online reviews MADM，且没有外部标签、公认排序或可复现实验指标，那么模型即使“发现”了复杂结构，也很难证明其正确性。这会重新回到“排序一致性”和“解释差异”的自说自话。

因此，这一研究线更适合放在有标签的多准则 sorting / ranking / classification benchmark 上，例如：

- QS / BCUR 等大学排名数据；
- Polish research units；
- monotone classification benchmark datasets；
- wine quality 等公开多属性分级数据；
- 其他具有明确类别标签或排序标签的公共数据集。

## 4. 可自适应发现的对象

当前构想中，“自适应发现”不应停留在抽象层面，而应具体化为以下可学习对象：

1. 准则单调性  
   自动判断某个准则应保持单调递增、单调递减，还是允许 U 型、倒 U 型或局部非单调。

2. 边际价值函数形状  
   自动选择线性、分段线性、spline、非单调函数或其他受约束函数形式。

3. 准则交互关系  
   在稀疏约束下自动发现少数具有实质贡献的 pairwise interactions，而不是人工指定交互项。

4. 类别阈值或排序边界  
   从 assignment examples 或 ranking labels 中学习 class boundary、preference threshold、indifference threshold 等参数。

5. 模型复杂度  
   自动判断某个数据集是否只需要简单 additive model，还是需要非单调项、交互项或更复杂的神经模块。

6. 何时不该复杂  
   一个真正有价值的自适应框架不应总是增加模块，而应能识别复杂模型何时会过拟合、解释性下降或计算代价过高。

## 5. 与现有文献的关系

现有文献已经分别解决了若干局部问题：

| 文献方向 | 已经做到的事情 | 对我们的启发与限制 |
|---|---|---|
| 非单调偏好学习 | Liu et al. (2019) 已经用 regularization 学习 potentially non-monotonic marginal value functions，并在 BCUR 大学排名数据上显著优于 UTADIS。 | “学习非单调”不是空白。我们的重点不能只是非单调，而应是何时允许非单调、何时保持单调。 |
| valued assignment learning | Liu et al. (2020) 已经处理 valued assignment examples，并比较多种 marginal value function。 | “更灵活的边际函数”已有基础，但仍主要处于 additive value model 体系内。 |
| interacting criteria | Liu et al. (2020, INFORMS JOC) 已经在 sorting 中学习准则交互，并用 regularization 控制复杂度。 | “学习交互项”不是空白。问题在于交互项如何稀疏选择、如何避免计算爆炸、如何与非单调结构共同建模。 |
| deep preference learning | EJOR 2023 已经使用神经网络进行 MCDA preference learning。 | “用深度学习做 MCDA”不是新颖主张，必须强调解释性、结构可控性和 benchmark 诊断。 |
| large-scale non-monotonic ML sorting | Omega 2025 已经探索大规模 assignment examples 下的非单调机器学习排序。 | 如果我们走神经网络路线，需要避免与该方向正面重复。 |
| attention-based interaction sorting | Information Fusion 2026 已经把 attention network 与 threshold-based sorting procedure 结合，用于捕捉准则交互。 | “用 attention 学交互”本身创新风险很高。 |
| temporal criteria preference learning | Li et al. (2025) 将时间序列准则与 monotonic RNN 结合。 | 时间维度是可扩展方向，但私有数据较多，不适合作为当前主 benchmark。 |

因此，当前更有机会的缺口不是某一个单独模块，而是：

> 现有方法往往预先选定一种偏好结构，例如单调、非单调、交互、深度聚合或 attention 交互；而我们希望研究的是偏好结构本身能否在 benchmark 证据下被自适应选择、验证和压缩。

## 6. 可能的论文贡献表达

### 贡献 1：自适应偏好结构选择

提出一个 adaptive preference-structure selection framework，将传统 MADM 中由研究者人工设定的结构假设转化为可学习对象，包括准则单调性、边际函数形状、交互项、阈值与复杂度。

### 贡献 2：可解释、受正则约束的结构学习模型

模型不追求纯黑箱复杂性，而是输出可审计的结构结果，例如：

- 每个准则的边际价值函数；
- 被保留或被拒绝的非单调形状；
- 被选择的准则交互项；
- 类别阈值或排序边界；
- 不同结构模块对 performance 的贡献。

### 贡献 3：跨 benchmark 的诊断式验证协议

不只报告一个最终 accuracy，而是比较：

- 简单 additive model；
- 单调模型；
- 非单调模型；
- 交互模型；
- 非单调 + 交互模型；
- 自适应结构选择模型；
- 可能的 neural / attention baseline。

验证指标包括：

- accuracy / F1 / recall / precision；
- Kendall tau 或 ranking consistency；
- calibration；
- ablation gain；
- 结构复杂度；
- 运行时间；
- 跨数据集稳定性；
- 解释性案例分析。

## 7. 当前可形成的研究问题

一个更紧的研究问题可以表述为：

> Can a multi-criteria sorting model automatically discover compact and interpretable preference structures, including monotonicity, non-monotonicity, criterion interactions, and class thresholds, while improving benchmark performance across heterogeneous decision scenarios?

中文表述：

> 多准则分级/排序模型能否在不依赖人工预设偏好结构的情况下，自动发现紧凑且可解释的偏好结构，并在异质 benchmark 场景中获得稳定提升？

## 8. 当前不应过度声称的内容

为了避免创新点虚化，当前不应声称：

- 我们是第一个将 AI 引入 MADM 的研究；
- 我们是第一个学习非单调偏好的研究；
- 我们是第一个学习准则交互的研究；
- 我们是第一个使用神经网络、attention 或 agent 思想的研究；
- 我们提出了一个通用到所有决策问题的万能框架；
- 偏好结构可以无约束地“自发涌现”。

更稳妥的声称是：

> 我们提出一种验证门控下的偏好结构自适应发现框架，使模型能够在多个有标签多准则 benchmark 中自动选择合适的偏好结构，并在准确性、解释性和复杂度之间取得更稳的平衡。

## 9. 下一步最小证据对象

下一步不应急于写完整模型，而应先构造一个 evidence packet：

1. 选定 4-6 个可复现 benchmark；
2. 为每个 benchmark 标注任务类型、准则数、样本量、标签来源、是否单调、是否可能存在交互；
3. 复现或整理现有代表方法的结果；
4. 建立一个最小结构搜索原型：
   - additive baseline；
   - monotone constrained model；
   - non-monotone marginal model；
   - sparse interaction model；
   - adaptive selection model；
5. 对比不同结构在不同 benchmark 上的表现；
6. 观察是否真的存在“不同场景需要不同偏好结构”的证据。

如果这个证据成立，文章故事会从“我们提出一个新模型”变成：

> 现有 MADM 方法的核心问题不是某个公式不够复杂，而是偏好结构被研究者预设；我们通过 benchmark 证据证明，不同任务需要不同结构，并提出一个能自动发现这些结构的可解释框架。
